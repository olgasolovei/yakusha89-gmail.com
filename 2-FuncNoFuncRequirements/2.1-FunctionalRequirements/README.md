# Таблиця 1. Функціональні вимоги

| ID          | Назва вимоги                          | Опис                                                                                                                                                                        |
| ----------- | ------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| FR1 / FR-R1 | Моніторинг ризику обриву тросу/канату | Система повинна відслідковувати стан тросів/канатів під час підйому вантажу та визначати потенційний ризик розриву на основі навантаження, зносу та частоти використання.   |
| FR2 / FR-R2 | Інформування працівників              | У разі виявлення високого ризику система повинна негайно повідомляти працівників через звуковий сигнал, світловий індикатор та/або push-повідомлення на мобільний пристрій. |
| FR3 / FR-R3 | Логування подій ризику                | Система повинна зберігати всі випадки перевищення допустимих навантажень та потенційного розриву тросів у базі даних для подальшого аналізу та формування звітів.           |
| FR4 / FR-R4 | Аварійне відключення                  | Система повинна автоматично зупиняти підйом вантажу при виявленні високого ризику розриву тросу/канату, щоб запобігти падінню вантажу.                                      |
| FR5         | Формування та відображення звітів     | Користувач має мати можливість формувати звіти та переглядати їх у структурованому вигляді з фільтруванням та доступом ≤3 кліки.                                            |
| FR6         | Налаштування та стійкість до збоїв    | Користувач може змінювати налаштування, які зберігаються автоматично; система має бути стійкою до збоїв і не втрачати дані.                                                 |
